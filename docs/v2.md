# Version 2 (v2)

**Status:** target specification — next milestone after [`docs/v1.md`](v1.md).

**Depends on:** v1 codebase (ingest, runtime, HLS sink machinery). v2 **replaces**
v1 manifest and source/sink wiring; it does **not** extend v1 manifests in place.

**North star:** [`docs/v7.md`](v7.md). v2 adds the **typed packet + inputs-only
sink** foundation that v7 remux and later decode/compose paths build on.

**Design references:** [`docs/source.md`](source.md) (packet model, abstract
streams). Earlier docs with sink `source_id` or v1 flat wiring are superseded for
v2 by **this file**.

---

## Reference scenarios (coverage, not a cap)

v2 integration proof is **typed ingest → filtered subscription → HLS remux** via
**sink `inputs[]` only** (no top-level `source_id` on sinks).

| Scenario | Description |
|----------|-------------|
| **`typed_remux_single_source`** | One publisher, video + voice → one remux sink |
| **`two_source_record`** | Two publishers, two remux sinks (each sink: inputs from one publisher) |
| **`cross_source_remux`** | Two publishers, **one** remux sink — e.g. video from A, voice from B |

Scenario graphs are **minimum tests**, not maximum size. v2 supports unbounded
`sources` and `sinks` with the same rules. A sink may bind inputs from **any
number of publishers** (including one input from one source only).

### typed_remux_single_source

```text
Publisher (RTMP or WHIP) → live/studio
  └── rec-a (HLS remux)
        inputs:
          kind: source, id: live/studio, video/h264, video/main
          kind: source, id: live/studio, voice/aac, voice/main
        → ./record/studio/.../stream.m3u8
```

### two_source_record

```text
live/studio → rec-a  (video/h264 + voice/aac — both inputs id: live/studio)
live/guest  → rec-b  (video/h264 + voice/opus — both inputs id: live/guest)
```

### cross_source_remux

```text
live/studio  ──video/h264──┐
                           ├──► rec-mux  → one HLS program
live/guest   ──voice/aac───┘
```

Example manifest input mix:

```json
{
  "sink_id": "rec-mux",
  "output_root": "./record/muxed",
  "mode": "record",
  "inputs": [
    {
      "kind": "source",
      "id": "live/studio",
      "stream_type": "video",
      "packet_type": "video/h264",
      "stream_id": "video/main"
    },
    {
      "kind": "source",
      "id": "live/guest",
      "stream_type": "voice",
      "packet_type": "voice/aac",
      "stream_id": "voice/main"
    }
  ]
}
```

The **muxer** picks a **main** input timeline and **remaps** other inputs’
PTS/DTS to match — **no A/V sync** in v2.

No decoders, composes, encoders, or events in v2 scenarios.

Full v7 graph: [`docs/scenarios/complex_two_source.md`](scenarios/complex_two_source.md).

| Entity | v2 rule |
|--------|---------|
| **Sources** | Unbounded `source_id` entries |

| **Decoders / composes** | **Not in v2 manifest** — deferred to v3+ |

## v2 in one paragraph

**Publish-only** RTMP and WHIP ingest assigns every sample **`stream_type`**,
**`packet_type`**, and **`stream_id`** (including metadata/data tracks). Sources
buffer typed packets on a shared presentation timeline. **HLS remux sinks** declare
**`inputs[]` only** (`kind: source`, publisher id, stream filter) — no
`source_id` on the sink object. Inputs may reference **the same or different**
publishers; remux accepts **`video` and `voice` only**. A dedicated **muxer
module** (separate from sink) handles multi-input interleave — including
cross-source programs — via **main-lane PTS remap only**. Manifest requires **`manifest_version: 2`**; v1 manifests
are rejected. Decode and compose graphs are out of scope.

---

## Architectural pillars

| Pillar | v2 rule |
|--------|---------|
| **Manifest** | **`manifest_version: 2` required** — reject missing, wrong, or v1 bodies |
| **Packet model** | Every stored sample has `stream_type`, `packet_type`, `stream_id`, pts/dts |
| **Abstract streams** | `stream_type` is the routing family — extensible (`video`, `voice`, `text`, `location`, …) |
| **Source buffer** | Push ingest → per-`stream_id` rings + shared timeline (see `source.md`) |
| **Subscription** | **Filtered pull** — reader declares stream filter; only matching packets |
| **Sink** | **HLS output only** — **`inputs[]` only**; **no `source_id`**; delegates mux to **Muxer** |
| **Muxer** | **Separate module** — main-timeline PTS remap + interleave; **no `av_sync`** |
| **Remux inputs** | **`stream_type` must be `video` or `voice`** — reject others on remux sinks |
| **Multi-source sinks** | **Allowed** — inputs may reference different `id` (cross-source mux) |
| **Wiring** | **Inputs only** — sinks never declare downstream; sources never declare sinks |
| **Decoders / composes** | **Out of v2** — not valid in v2 manifest |

## Packet model and abstract streams

### Fields (every `SourcePacket`)

| Field | Role | Examples |
|-------|------|----------|
| **`stream_type`** | Abstract stream family — decode/remux/routing boundary | `video`, `voice`, `text`, `location` |
| **`packet_type`** | Payload format (MIME-like) | `video/h264`, `voice/aac`, `voice/opus`, `text/prompt`, `location/text` |
| **`stream_id`** | Track within one publisher | `video/main`, `voice/main`, `text/prompt`, `meta/location` |
| **`pts` / `dts`** | Session-comparable timestamps | |
| **`payload`** | Opaque bytes | |

### Derivation rules

- **`stream_type`** is derived from **`packet_type`** prefix when not explicitly
  set at ingest (`video/h264` → `video`; `voice/aac` → `voice`;
  `location/text` → `location`; `text/prompt` → `text`).
- Ingest **must** assign consistent triples; manifest filters match on declared
  fields on each sink input.
- **`stream_id`** distinguishes parallel tracks of the same `stream_type` (e.g.
  `voice/main` and `voice/instruction`).

### v2 ingest mapping (RTMP / WHIP)

| Ingest | v2 behavior |
|--------|-------------|
| **RTMP** FLV video tag | `stream_type: video`, `packet_type: video/h264`, `stream_id: video/main` (default ids; configurable later) |
| **RTMP** FLV audio tag | `stream_type: voice`, `packet_type: voice/aac`, `stream_id: voice/main` |
| **RTMP** metadata / data | Map to `text/*`, `location/*`, or other `stream_type` per metadata keys — **stored and subscribable**, not remuxed in v2 |
| **WHIP** H264 | `video/h264`, `video/main` |
| **WHIP** Opus | `voice/opus`, `voice/main` |

v2 **must** ingest and buffer non-A/V abstract streams when the publisher sends
them. v2 remux sinks **must not** list them in `inputs[]` (validation rejects).

### Storage (target alignment with source.md)

- Per-`stream_id` ring buffers + shared presentation timeline.
- Subscriptions walk the timeline (or per-stream rings with filter) and return
  only packets matching the consumer's filter.

v1 single-queue FLV tag stream is **replaced** by typed storage in v2 implementation.

## Subscriptions (filtered pull)

### Filter spec

Each consumer (remux sink worker) attaches with a filter matching one sink input:

```text
SubscriptionFilter
  source_id          string     publisher id (from input.id)
  stream_type        string     e.g. video, voice
  packet_type        string     e.g. video/h264
  stream_id          string     e.g. video/main
```

All four fields on the manifest input **must match** the packet for delivery.

### API (illustrative)

```text
ISource
  ├── WaitReady(SourceFormat*)     → stream bundle / track catalog
  ├── Subscribe(SubscriptionFilter) → ISourceSubscription*
  └── Close()

ISourceSubscription
  ├── Read(SourcePacket*)          → blocks until matching packet or close
  └── Close()
```

One remux sink with N inputs creates **N filtered subscriptions** (possibly on
**different sources**). The **muxer** reads from those subscriptions and
produces ordered mux units; the **sink** writes HLS output.

## Muxer module (separate from sink)

v1 folded demux/mux/interleave into `HlsSinkRunner` + `hls_sink.cpp`. v2 **splits**
responsibilities:

```text
SinkInput[]  →  Muxer  →  SinkWriter (HLS)
     ↑              ↑            ↑
 filtered      sync +         segments,
 subscriptions  interleave     playlist, mode
```

### Why separate

| Concern | Owner |
|---------|--------|
| Which streams, which publishers | Manifest + runtime wiring |
| Per-input filtered `Read` | Subscription layer |
| **Multi-input ordering, main-lane PTS remap, interleave** | **Muxer** |
| Segment boundaries, playlist, record/live policy | **Sink** (HLS writer) |

Cross-source remux (video from A, voice from B) requires timeline coordination
that does not belong in “write HLS files” code. v2 uses a **simple main-timeline
remap** — not proper A/V sync (see below).

### Timeline policy (v2 — no A/V sync)

v2 muxer **does not** implement `av_sync`, lip-sync, drift correction, or
wall-clock alignment. It uses a **single main timeline** and **rewrites** other
lanes to match.

**Rules (locked):**

1. **Choose one main lane** at muxer `Prepare` time:
   - **Default:** first **`stream_type: video`** input in manifest `inputs[]` order.
   - If no video input (voice-only sink — if allowed): first input is main.
2. **Main lane** — PTS/DTS pass through unchanged (output timestamps = main lane
   timestamps).
3. **Secondary lanes** — on each packet, **replace** PTS/DTS so the sample aligns
   to the main timeline:
   - Compute offset once at lane start:  
     `offset = main_lane_first_pts - secondary_lane_first_pts`
   - For each secondary packet:  
     `out_pts = in_pts + offset`, same for DTS (or derive DTS from PTS if missing).
4. **Interleave** by the **remapped** PTS/DTS (main + adjusted secondaries).
5. **No drift handling** — if secondary source runs faster/slower, timestamps
   are still linearly offset only; gaps/overlaps are not corrected.

```text
Main (video @ studio):  pts 0, 33, 66, ...     → written as-is
Secondary (voice @ guest): pts 1000, 1020, ... → rewritten to 0, 20, ... (after offset)
```

This is sufficient for v2 **container mux** (HLS accepts the timestamps written).
Operators must not expect broadcast-quality sync from cross-source remux in v2.

**Deferred to later versions:** `av_sync`, adaptive drift, silence padding, drop/skip
to maintain lip-sync, RTCP/ wall-clock correlation.

### Muxer responsibilities (v2)

1. Accept **N input lanes** — each lane = one manifest sink input + filtered subscription.
2. **Select main lane** (video-first rule above).
3. **Pull** typed packets from each lane.
4. **Remap PTS/DTS** on non-main lanes (offset from first seen packet per lane).
5. **Interleave** by remapped timestamps.
6. Emit **mux packets** (or AVFormat write calls) to the sink writer.

### Muxer does not

- Parse RTMP/WHIP (ingest)
- Own `output_root` or HLS segment retention
- Know HTTP manifest or runtime reconcile
- **`av_sync`** — lip-sync, drift compensation, or wall-clock A/V alignment

### Sink responsibilities (v2)

1. Own **`output_root`**, **`mode`** (record/live), sink lifecycle thread.
2. Initialize HLS container from muxer stream descriptors (codecs, extradata from first packets).
3. Receive interleaved output from **Muxer** and write segments + playlist.
4. Close cleanly when inputs end or runtime removes sink.

### Module layout (target)

```text
core/
  mux/
    imuxer.h              abstract mux interface
    remux_muxer.cpp       multi-input encoded remux (v2)
  sinks/
    hls_sink_writer.cpp   segment/playlist only (no multi-source logic)
    sink_handle.cpp       wires inputs → RemuxMuxer → HlsSinkWriter
```

`HlsSinkRunner` from v1 is **refactored**: mux logic moves to `RemuxMuxer`; runner
becomes thin orchestration or is replaced by `SinkHandle` + muxer + writer.

### Public mux API (illustrative)

```cpp
struct MuxInputLane {
  std::string source_id;
  SubscriptionFilter filter;
  ISourceSubscription* subscription;
};

class IRemuxMuxer {
 public:
  virtual bool Prepare(const std::vector<MuxInputLane>& lanes) = 0;
  virtual void Close() = 0;
  // Main lane index chosen in Prepare; secondary PTS/DTS remapped, no av_sync
  virtual bool Step(MuxStep* out) = 0;
};
```

Sink loop (conceptual):

```cpp
while (muxer->Step(&step)) {
  hls_writer->Write(step);
}
```

## Sinks (v2)

Remux **sink** = manifest binding + **RemuxMuxer** + **HLS writer**. Timeline
remap and interleave live in **Muxer**, not in sink/HLS code.

### Remux sink — inputs only

Sinks have **`inputs[]`**. There is **no `source_id`** field on the sink.

```json
{
  "sink_id": "rec-a",
  "output_root": "./record/studio",
  "mode": "record",
  "inputs": [
    {
      "kind": "source",
      "id": "live/studio",
      "stream_type": "video",
      "packet_type": "video/h264",
      "stream_id": "video/main"
    },
    {
      "kind": "source",
      "id": "live/studio",
      "stream_type": "voice",
      "packet_type": "voice/aac",
      "stream_id": "voice/main"
    }
  ]
}
```

Guest publisher (Opus):

```json
{
  "sink_id": "rec-b",
  "output_root": "./record/guest",
  "mode": "record",
  "inputs": [
    {
      "kind": "source",
      "id": "live/guest",
      "stream_type": "video",
      "packet_type": "video/h264",
      "stream_id": "video/main"
    },
    {
      "kind": "source",
      "id": "live/guest",
      "stream_type": "voice",
      "packet_type": "voice/opus",
      "stream_id": "voice/main"
    }
  ]
}
```

### Sink fields

| Field | Required | Notes |
|-------|----------|-------|
| `sink_id` | yes | unique |
| `output_root` | yes | HLS output root |
| `mode` | no | `"record"` (default) or `"live"` |
| `inputs` | yes | non-empty array |

### Sink input fields (v2)

| Field | Required | Notes |
|-------|----------|-------|
| `kind` | yes | **`source` only** in v2 |
| `id` | yes | must exist in manifest `sources[]` |
| `stream_type` | yes | remux: **`video` or `voice` only** |
| `packet_type` | yes | must be consistent with `stream_type` prefix |
| `stream_id` | yes | track id within publisher |

### Validation rules

1. **`inputs` non-empty** — reject empty array.
2. **`kind` must be `source`** — reject other kinds (compose/decoder deferred).
3. **Multi-source allowed** — inputs may reference **different** `id` values on the same sink (cross-source mux).
4. **Remux stream types** — each input `stream_type` must be **`video` or `voice`**; reject `text`, `location`, etc.
5. **`packet_type` / `stream_type` consistency** — e.g. `video/h264` requires `stream_type: video`; reject mismatch.
6. **Source reference** — each `id` must appear in `sources[]`; strict upsert requires each referenced source **active** when starting routes.
7. **No duplicate input** — same `(id, stream_id, packet_type)` twice on one sink → reject.
8. **At least one `video` input** recommended for HLS remux — reject video-less sinks if HLS requires video (implementation: reject mux with zero video lanes).

### HLS behavior

- Same record/live modes as v1 (`playlist_size`, segment retention).
- Output path: sink chooses output identity (e.g. primary video input’s `{app}/{stream}` or synthetic `mux/{sink_id}` — **TBD at implement**, document in code).
- **Muxer** interleaves; **sink writer** only writes segments/playlist.
- FLV tag path retired for v2 sink wiring — muxer consumes **typed packets** only.

### Encode-branch / event sinks

**Out of v2** — no `encoder_id`, no `sink_kind: event_jsonl`.

## Runtime and HTTP (v2)

### Apply manifest

Same lifecycle as v1 (`ApplyManifest`, bootstrap vs strict upsert, desired-state
diffing) with v2 validation and typed subscription wiring.

On manifest apply / source accept:

1. For each desired sink, resolve inputs → filtered subscriptions (possibly multiple sources).
2. Construct **RemuxMuxer** with N lanes.
3. Construct **HlsSinkWriter** with `output_root` + `mode`.
4. Start sink worker thread: `muxer->Step` → `writer->Write`.

If a cross-source sink references a source not yet active, start sink when **all**
referenced sources are active (or start partial and block lanes — **prefer wait
for all inputs** before first segment).

Removing a source from manifest still does not disconnect an active publisher
(v1 behavior; may improve in later versions).

### HTTP API

| Endpoint | Methods | v2 |
|----------|---------|-----|
| `/upsert_manifest` | `POST` | v2 JSON only |
| `/upsert_manifest` | `OPTIONS` | CORS preflight |

**Out of v2:** `GET /status`, `GET /compose_types`, manifest read-back.

Response shape unchanged: `{ "ok": true }` or `{ "ok": false, "error": "..." }`.

## CLI (v2)

CLI must emit **`manifest_version: 2`** and sink **`inputs[]`** (no `--sink`
without building typed inputs from the current `--source`).

Minimum v2 bootstrap example:

```text
./argus --source live/studio \
  --sink ./record/studio --input video/h264:video/main --input voice/aac:voice/main
```

Exact flag design is implementation detail; v2 CLI **must not** produce v1
flat sinks with top-level `source_id` only.

**Out of v2 CLI scope:** `--decoder`, compose flags (deferred with v3+ manifest).

## Definition of done (v2)

### Manifest

- [ ] `manifest_version: 2` enforced; v1 bodies rejected
- [ ] `decoders` / `composes` rejected if present
- [ ] Sink `inputs[]` required; sink `source_id` rejected
- [ ] Remux inputs: `video` / `voice` only; **multi-source inputs allowed**
- [ ] `packet_type` / `stream_type` consistency validated

### Muxer module

- [ ] **`RemuxMuxer`** (or equivalent) separate from HLS writer
- [ ] Single-source multi-input mux (video + voice, same publisher)
- [ ] **Main-lane timeline** — first video input is main; secondaries get PTS/DTS offset remap
- [ ] **No `av_sync`** — document in tests that drift is not corrected
- [ ] Unit tests for interleave + offset remap (mock lanes)

### Ingest + source

- [ ] RTMP assigns typed packets for video, audio, metadata/data tracks
- [ ] WHIP assigns typed packets for H264 + Opus
- [ ] Non-A/V streams buffered and visible in stream bundle
- [ ] Filtered subscriptions deliver only matching packets

### Sinks

- [ ] HLS sink = **muxer + writer** (no mux logic in writer)
- [ ] Record and live modes
- [ ] Multi-sink fan-out; multi-source per sink

### Scenarios

- [ ] **`typed_remux_single_source`** integration test passes
- [ ] **`two_source_record`** integration test passes
- [ ] **`cross_source_remux`** integration test passes

### Tests

- [ ] Unit tests: manifest validation (version, inputs, stream types)
- [ ] Unit tests: packet_type / stream_type consistency
- [ ] Go or C++ integration: publish → typed remux → HLS playable

| # | Decision |
|---|----------|
| 1 | Sinks **`inputs[]` only** — **no `source_id`** on sink |
| 2 | Input `kind: source` only in v2 |
| 3 | **`manifest_version: 2` required** — no v1 support |

## Assumptions

- Publishers use `source_id` = `{app}/{stream}` (RTMP/WHIP unchanged).
- Default `stream_id` names (`video/main`, `voice/main`) until manifest/config overrides exist.
- HLS remux remains the only sink implementation in v2.
- Cross-source sinks require **all referenced sources active** before mux starts (v2 policy).
- Muxer **main lane** = first video input; other lanes get **static PTS/DTS offset** from first packet — **no drift / av_sync**.

---

## Summary

v2 is **typed remux**: **`manifest_version: 2`**, abstract **`stream_type`**
streams on every packet, **filtered pull** subscriptions, a separate **Muxer
module** for multi-input and **cross-source** interleave (**main-lane PTS remap,
no `av_sync`**), and **HLS sinks**
that declare only **`inputs[]`** (no `source_id`) with **`video` + `voice`**
lanes. Metadata and data tracks are ingested for later graph nodes; decode and
compose are deferred. **`typed_remux_single_source`**, **`two_source_record`**, and
**`cross_source_remux`** prove the milestone end-to-end.
