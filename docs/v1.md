# Version 1 (v1 — as-built)

**Status:** documents **what the codebase implements today** (`core/`, `cmd/argus/`).

**Relationship to v7:** [`docs/v7.md`](v7.md) is the **north star** (v_latest). v1 is the
**current shippable baseline** — a strict subset of v7 capability. Future versions
(v2…v7) are defined by **adding** capability on top of v1, not by rewriting v1.

**This file wins** for “what does v1 do?” Design docs (`docs/compose.md`,
`docs/pipeline.md`, README manifest examples with `inputs[]`) describe **targets**
beyond v1 unless they match the structs in `runtime_manifest.h`.

**Code namespace:** `irs3` (legacy internal name). Binary: **`argus`**.

---

## Reference scenario (coverage, not a cap)

v1 integration proof is **ingest → buffer → HLS remux**, optionally **H264 decode →
terminal composes**. Typical test graph:

```text
Publisher (RTMP or WHIP)
  └── source live/test
        ├── sink record-a  → ./record-a/.../stream.m3u8  (record)
        ├── sink live-b    → ./live-b/.../stream.m3u8     (live, optional)
        └── decoder video-dec-a
              ├── compose snap-a   (jpg_snapshot → ./frames/)
              ├── compose layout-a (side_by_side → ./scenes/)
              └── compose score-a  (clip_prompt → ./infer/infer.log)
```

Go tests under `core/tests/` exercise RTMP publish, WHIP, multi-server, and HTTP
`POST /upsert_manifest`. v1 supports **multiple sources** in one manifest; the
scenario above uses one publisher as minimum coverage.

---

## Scale (no artificial limits in v1 schema)

v1 manifest imposes **no fixed maximum** on entities the runtime can hold:

| Entity | v1 rule |
|--------|---------|
| **Sources** | Unbounded `source_id` entries; one active publisher per id |
| **Sinks** | Unbounded per source — fan-out via independent subscriptions |
| **Decoders** | Unbounded per source — each gets its own source subscription |
| **Composes** | Unbounded per decoder — all attach as push consumers on same decoder |

Limits are **operational** (CPU, disk, ONNX load for `clip_prompt`), not schema.

---

## v1 in one paragraph

**Publish-only** RTMP and WHIP ingest into per-session **`BufferedSource`** ring
buffers; **HLS remux sinks** pull encoded media and write record or live playlists;
optional **H264 decoders** pull the same sources on separate subscriptions and
**push** decoded frames to **terminal compose plugins** (`jpg_snapshot`,
`side_by_side`, `clip_prompt`). Graph state is declared via **CLI bootstrap** or
**HTTP `POST /upsert_manifest`** (full desired-state replace with diffing). No
encoders, no events, no compose DAG — composes bind to **`decoder_id`** only.

---

## Architectural pillars

| Pillar | v1 rule |
|--------|---------|
| **Ingest** | RTMP (FLV tags) + WHIP (packetized H264/Opus); publish-only |
| **Source buffer** | Push ingest → pull read via **`ISourceSubscription::Read`** |
| **Sink** | **HLS remux only** — one output program per sink, bound to **`source_id`** |
| **Decoder** | **H264 video only** — one decoder entry per manifest row, bound to **`source_id`** |
| **Compose** | **Three concrete types** — wired by **`decoder_id`**, **push** `OnVideoFrame` |
| **Manifest** | `sources`, `sinks`, optional `decoders`, optional `composes` — **no `manifest_version`** |
| **Control plane** | `POST /upsert_manifest` only; bootstrap vs strict active-source validation |
| **Wiring** | Flat: source → sink; source → decoder → compose(s). **No `inputs[]` graph** |

## Sources

### Ingest servers

Both servers implement **`IServer`**, push accepted **`ISource`** instances to
**`SourceQueue`**, consumed by **`Runtime::HandleSource`**.

| Server | Protocol | Source id | Payload mode |
|--------|----------|-----------|--------------|
| **RTMP** | `SourceProtocol::kRTMP` | `{app}/{stream}` e.g. `live/test` | `kFlvTag` — FLV tag types 8/9/18 |
| **WHIP** | `SourceProtocol::kWHIP` | `{app}/{stream}` from path `/{app}/{stream}/whip` | `kPacketized` — H264 + Opus |

**RTMP** (`core/servers/rtmp/rtmp_server.cpp`):

- One session per publisher connection
- Audio/video/data reconstructed as FLV tags into `BufferedSource`
- GOP resume: metadata + A/V config tags stored; subscribers replay prefix on lag

**WHIP** (`core/servers/webrtc/webrtc_server.cpp`):

- POST SDP offer, DELETE with session id
- Video: H264 only; audio: Opus only
- RTP depacketized → `SourcePacket` with pts/dts

**CLI note:** `--server rtmp|webrtc|both` is parsed in `main.cpp` but **ignored** —
the process **always** starts both RTMP (default port 1935) and WHIP (default 8080).

Server config structs still carry `output_root`; **`main.cpp` clears it** — all
output paths come from manifest sinks / compose `output_root`.

### Packet model (`core/sources/source_packet.h`)

```text
SourcePayloadMode     kFlvTag | kPacketized
SourceTrackKind       kVideo | kAudio | kData
SourceCodec           kH264 | kAAC | kOpus | kUnknown

SourceFormat          payload_mode + tracks[] (SourceTrackConfig)
SourcePacket          sequence, stream_index, track_kind, tag_type,
                      timestamp_ms, pts, dts, duration, key_frame,
                      gop_start, shared payload vector
```

v1 does **not** expose `packet_type` / `stream_id` strings in manifest — routing
is implicit (first H264 video in decoder path; full mux in HLS sink).

### Buffered source and subscriptions (`core/sources/buffered_source.h`)

```text
ISource
  ├── Descriptor()           → SourceDescriptor { id, protocol, app, stream, session_id }
  ├── WaitReady(SourceFormat*)
  ├── Subscribe()            → ISourceSubscription
  └── Close()

ISourceSubscription
  ├── Read(SourcePacket*)    → blocks until packet or close
  └── Close()
```

- Ring buffer default **4096** packets per source
- Each sink/decoder gets an **independent subscription** (fan-out)
- Sequence-based read; on lag, rewinds via **GOP prefix** packets

### Source acceptance (`Runtime::HandleSource`)

1. `descriptor.id` must exist in **desired** manifest `sources`
2. Rejects duplicate active `source_id`
3. Starts all desired sinks, decoders, composes for that source

Removing a source from manifest does **not** tear down an already-active publisher
(v1 limitation).

## Sinks (v1)

**Only sink type:** HLS remux (`SinkHandle` → `HlsSinkRunner` → `irs3_hls_sink`).

### Manifest fields

```json
{
  "sink_id": "record-a",
  "source_id": "live/test",
  "output_root": "./record-a",
  "mode": "record"
}
```

| Field | Required | Values |
|-------|----------|--------|
| `sink_id` | yes | unique string |
| `source_id` | yes | must reference manifest source |
| `output_root` | yes | directory root for HLS tree |
| `mode` | no | `"record"` (default) or `"live"` |

No `inputs[]`, `sink_kind`, or `encoder_id` in v1.

### Behavior

- **Pull** from source subscription on dedicated **`HlsSinkRunner`** thread
- **RTMP sources:** FLV demux inside `hls_sink.cpp`, then HLS mux
- **WHIP sources:** direct elementary stream write (H264/AAC/Opus)
- **Record mode:** `playlist_size = 0`, segments retained
- **Live mode:** sliding playlist (`playlist_size = 3`, extra segment retention)

Output path pattern (`hls_sink.cpp`):

```text
{output_root}/{app}/{stream}/{app-stream-sessionId}/stream.m3u8
```

Example: `./record-a/live/test/live-test-12345/stream.m3u8`

## Decoders (v1)

### Manifest fields

```json
{
  "decoder_id": "video-dec-a",
  "source_id": "live/test"
}
```

Only **`decoder_id`** and **`source_id`**. No `stream_type`, `packet_type`, or
track selection — always **H264 video** from the source subscription.

### Pipeline

```text
DecoderHandle
  └── DecoderRunner (thread)
        ├── subscription->Read(SourcePacket)
        ├── AdaptH264SourcePacket (FLV or packetized)
        ├── H264Decoder::FeedPacket / TryDecode
        └── DispatchFrame → IDecodedVideoConsumer::OnVideoFrame
```

- **`H264Decoder`:** FFmpeg `AV_CODEC_ID_H264`, AVCC BSF + Annex-B fallback
- **Multiple compose consumers:** frame **cloned** per consumer when count > 1
- **No audio decode** in v1 (AAC/Opus pass through to HLS only)

## Compose (v1)

### Public interface (`core/compose/icompose.h`)

```text
IDecodedVideoConsumer
  └── OnVideoFrame(const VideoFrame&)

ICompose : IDecodedVideoConsumer
  ├── Prepare()
  └── Close()
```

**No** `ReadPacket`, **no** `SubscribeEvents`, **no** compose registry metadata API.

### Wiring (legacy push)

```text
ComposeHandle
  └── decoder_handle_->AddConsumer(compose_)
        └── OnVideoFrame pushed from DecoderRunner thread
```

Manifest binds compose → **`decoder_id`** (not `inputs[]`). Multiple composes
may share one decoder (parallel push consumers).

### Registered compose types

| `compose_type` | Class | Output | Params |
|----------------|-------|--------|--------|
| `jpg_snapshot` (default) | `JpgSnapshotCompose` | `{output_root}/frame_NNNNNN.jpg` | `snapshot_interval` (default 50) |
| `side_by_side` | `SideBySideCompose` | `{output_root}/scene_NNNNNN.jpg` | `snapshot_interval` |
| `clip_prompt` | `ClipPromptCompose` | `{output_root}/infer.log` + stderr | `prompt` (required), `model_root`, `snapshot_interval` |

Factory: `MakeCompose()` in `compose_handle.cpp`; validation via `IsKnownComposeType()`.

### Per-type behavior

**`jpg_snapshot`** — every Nth decoded frame written as JPEG via `jpg_writer.cpp`.

**`side_by_side`** — `BuildSideBySideFrame()` duplicates **the same frame** left
and right (demo layout, **not** two-source panel). Writes JPEG scene stills.

**`clip_prompt`** — MobileCLIP2-S2 ONNX inference; logs cosine similarity and
latency to stderr and `infer.log`. Default models: `./models/mobileclip2_s2`
(download via `tools/download_mobileclip2_s2_models.sh`). Requires
`brew install onnxruntime`.

All three sample by **`snapshot_interval`** decoded frames (default **50**).

### Manifest compose block

```json
{
  "compose_id": "snap-a",
  "compose_type": "jpg_snapshot",
  "decoder_id": "video-dec-a",
  "output_root": "./frames",
  "snapshot_interval": 50
}
```

```json
{
  "compose_id": "score-a",
  "compose_type": "clip_prompt",
  "decoder_id": "video-dec-a",
  "output_root": "./infer",
  "prompt": "a minion character",
  "model_root": "./models/mobileclip2_s2",
  "snapshot_interval": 50
}
```

**Validation:**

- `compose_id`, `decoder_id`, `output_root` required
- `decoder_id` must exist in manifest `decoders[]`
- `compose_type` must be known
- `clip_prompt` requires non-empty `prompt`
- `snapshot_interval > 0`

**Not supported:** compose → compose chain, compose reading source directly,
multi-input fan-in, media output back into graph.

## Runtime and manifest control

### Top-level manifest (v1)

```json
{
  "sources": [
    { "source_id": "live/test" },
    { "source_id": "live/guest" }
  ],
  "sinks": [ "..." ],
  "decoders": [ "..." ],
  "composes": [ "..." ]
}
```

**No `manifest_version` field** — parser ignores unknown top-level keys only
implicitly (not stored). No `encoders` array.

C++ structs: `RuntimeManifest` in `core/runtime/runtime_manifest.h`.

### Apply modes

| Mode | Trigger | `require_active_sources` |
|------|---------|--------------------------|
| **Bootstrap** | CLI `initial_manifest` at startup | `false` — routes allowed before publisher connects |
| **Strict upsert** | HTTP `POST /upsert_manifest` | `true` — sink/decoder `source_id` must be **actively publishing** |

**`ApplyManifest`** (`runtime.cpp`):

1. Validate manifest
2. Diff desired vs active — close changed/removed sinks, decoders, composes
3. Update `RuntimeDesiredState`
4. Start new routes for **already-active** sources
5. Full replace of desired state; graph unchanged on validation failure

Public API: `Runtime::ApplyManifest(manifest)` → `ApplyManifestResult { ok, error }`.

### Active state indexes (`runtime_state.h`)

Tracks: `active_sources`, `active_routes` (sinks), `active_decoder_routes`,
`active_compose_routes`, plus spec maps and desired-state indexes built by
`BuildRuntimeDesiredState()`.

## HTTP API (v1)

| Endpoint | Methods | Behavior |
|----------|---------|----------|
| `/upsert_manifest` | `POST` | JSON body → parse → strict `ApplyManifest` |
| `/upsert_manifest` | `OPTIONS` | CORS preflight |

Default bind: **`127.0.0.1:8090`** (`--runtime-port`).

Request: `Content-Type: application/json`

Response:

```json
{ "ok": true }
```

or

```json
{ "ok": false, "error": "..." }
```

**Not in v1:** `GET /compose_types`, health, manifest read-back, event streams.

Parser: `parse_manifest_json()` in `runtime_http.cpp` — fields listed in
[Manifest compose block](#manifest-compose-block) and sink/decoder sections above.

## CLI (`cmd/argus/main.cpp`)

Builds `RuntimeManifest`, passes as `RuntimeConfig.initial_manifest`, starts
RTMP + WHIP + runtime HTTP.

| Flag | Effect |
|------|--------|
| `--host` | Bind host (default `127.0.0.1`) |
| `--rtmp-port` | RTMP (default `1935`) |
| `--webrtc-port` | WHIP (default `8080`) |
| `--runtime-port` | HTTP (default `8090`) |
| `--source ID` | Append source; reset current decoder context |
| `--sink` / `--output-root PATH` | HLS sink for current source |
| `--live` | Next sink uses live mode |
| `--decoder` | Add decoder for current source |
| `--compose DIR` | `jpg_snapshot` (auto-creates decoder if needed) |
| `--compose-side-by-side DIR` | `side_by_side` |
| `--compose-infer-prompt "text" DIR` | `clip_prompt` |
| `--server` | Parsed, **not applied** |
| `--port` | **Rejected** (exit 1) — use `--rtmp-port` |

**Defaults:** if no flags, `live/test` → `./out/argus` record sink.

**Multi-source example:**

```text
./argus --source "live/cam-a" --sink ./record-a/ \
        --source "live/cam-b" --sink ./record-b/
```

**Record + decode + composes:**

```text
./argus --source "live/test" --sink ./record/ --decoder \
  --compose ./frames/ \
  --compose-infer-prompt "a minion character" ./infer/
```

Auto-generated ids: `{source_id}#{n}` for sinks, `{source_id}#decoder-{n}` for
decoders, `{decoder_id}#compose-{n}` for composes.

## Reference graph (typical v1 manifest)

```text
Sources:
  live/test, live/guest          (any count)

Per source (example live/test):
  sinks: record-a, live-b        (fan-out subscriptions)
  decoder: video-dec-a
    composes: snap-a, score-a   (parallel push consumers)

Per source (example live/guest):
  sinks: record-b                (remux only — no decode required)
```

Multi-source **works today** — each source/decoders/sinks validated independently.
This is **not** `complex_two_source.md` (no typed packet routing, encoders, compose DAG).

## Assumptions (documented)

- Publishers use source ids matching manifest (`app/stream` convention).
- H264 video is the only decode path; other codecs are remux-only.
- `clip_prompt` requires ONNX models on disk and ONNX Runtime installed.
- libdatachannel vendored under `third_party/libdatachannel` or parent fallback.
- One active RTMP/WHIP session per `source_id` at a time.

---

## Summary

v1 is the **as-built Argus runtime**: RTMP + WHIP ingest, buffered sources with
subscription fan-out, HLS record/live remux sinks, optional H264 decode, and
three **terminal push composes** controlled by a flat manifest (`sources`,
`sinks`, `decoders`, `composes`) via CLI or HTTP. It is the foundation v7 extends
with typed packets, compose graphs, encoders, and Argus watching (events).
